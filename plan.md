# Веб-интерфейс загрузки акционных наборов (Box) Pâté в FoodSave CRM

Создание веб-приложения для автоматической обработки WhatsApp-сообщений барист/менеджеров 5 точек Pâté, визуального контроля распарсенных позиций, авторизации в CRM и массового создания карточек товаров (`POST /api/products`).

---

## 1. Функциональные требования

### 1.1. Входные форматы сообщений (WhatsApp)
Система должна уметь распознавать все вариации сообщений с точек:

* **Формат 1 (Стандарт с единицами и адресом):**
  ```text
  Пате Бухар Жырау (Адрес тут)
  Слойка с черника 540т вместо 900т 8 шт
  Улитка с шоколадом 870т вместо 1450т 8шт
  ```

* **Формат 2 (Опечатки типа "вместе", отсутствие "шт", кол-во в начале/конце):**
  ```text
  Пате Туран 
  Синнабон 2шт 690 вместе 1150
  Улитка шоколадом 6шт 870 вместе 1450
  Тарт творожный макси 5160 вместо 8600
  ```

* **Формат 3 (Проценты в заголовке, нумерация, цены с точками, дефисы):**
  ```text
  Пате Достык: 40% 
  1. синнабон - 3 шт - 690 вместо 1150 
  10. тарт ягодный микс макс - 1 шт - 7200 вместо 12.000
  ```

### 1.2. Авторизация в CRM (`/api/auth/login`)
- Форма логина по email и паролю для получения JWT `accessToken` и `refreshToken`.
- Безопасное сохранение токена в `localStorage`.
- Автоматический refresh токена (`/api/auth/refresh-token`) при ошибке `401 Unauthorized`.
- Отображение статуса авторизации и активных списков точек (`GET /api/stores/active`) и категорий (`GET /api/categories/active`).

### 1.3. Парсинг и поддержка Google Gemini AI
- **Два режима парсинга (переключатель в UI):**
  1. **Правиловый/Fuzzy парсер (локальный JS/Python):**
     - Расширенные регулярные выражения для поддержки `"вместе"`, `"вместо"`, `"540т"`, `"12.000"`, нумерации `1.`, вариаций количества (`2шт`, `3 шт`).
     - Fuzzy-сопоставление названий с каталогом `catalog_paste.json`.
  2. **ИИ Парсер (Google Gemini AI API):**
     - Поле ввода Google Gemini API Key (сохраняется в `localStorage`).
     - Запрос к Gemini API (`gemini-2.5-flash`) с передачей сырого текста и базы каталога в виде JSON Schema.
     - 100% точность распознавания любых человеческих опечаток, нестандартных переносов и форматирования.

### 1.4. Визуальный контроль и редактирование предпросмотра
- Интерактивные карточки товаров, сгруппированные по заведениям (store).
- Бейджи статусов: `needsReview` (требует внимания), `score` совпадения с каталогом, размер (`мини`/`большой`/`порция`), скидка `%`.
- Предпросмотр изображения товара из каталога.
- Возможность вручную изменить название, цену, скидку, количество, заведение или сопоставление с каталогом до отправки.

### 1.5. Отправка в CRM и отчет
- Кнопка «Отправить в CRM».
- Режим `--dry-run` (проверка данных без сохранения в CRM).
- Пошаговый прогресс-бар отправки каждого товара через `POST /api/products`.
- Лог результатов (успешно создано / ошибки).


### [NEW] `server.py`
Легковесный сервер на Python (стандартная библиотека + proxy для CORS).
- Раздает веб-интерфейс `web/index.html`.
- Проксирует запросы к `https://foodsave.kz/api/` для избежания CORS ошибок в браузере.
- Предоставляет эндпоинты `/api/parse` (запуск парсинга) и `/api/upload` (загрузка в CRM при необходимости).

### [NEW] `web/index.html` & `web/app.js`
- **Шапка (Header):** Статус авторизации, логин/выход, индикаторы активных заведений CRM.
- **Панель входа:** Ввод Email / Password для авторизации в FoodSave CRM.
- **Панель парсинга:**
  - Текстовое поле ввода сообщений из WhatsApp.
  - Настройки: переключатель "Локальный парсер / Gemini AI", поле ввода Gemini API Key.
  - Кнопка "Обработать список".
- **Секция предпросмотра:**
  - Карточки товаров по точкам (Пате Туран, Пате Достык, Пате Бухар Жырау...).
  - Индикаторы скидки, цены, количества, срока годности, фото.
  - Редактирование любых полей прямо в карточке.
  - Кнопки: "Тестовая проверка (Dry Run)" и "Опубликовать в CRM".

### [MODIFY] `parse_daily.py`

## 3. Требуется подтверждение / Открытые вопросы

> [!IMPORTANT]
> **Интеграция Gemini AI**: Будем использовать прямое обращение к Gemini REST API из веб-интерфейса через API ключ пользователя (ключ хранится только в локальном браузере `localStorage`). Это удобно, бесплатно и не требует сторонних серверов.

> [!NOTE]
> **Авторизация**: Авторизация производится напрямую в FoodSave CRM (`https://foodsave.kz/api/auth/login`). После входа полученный токен автоматически подставляется во все запросы создания товаров `POST /api/products`.

## 4. План проверки

### Автоматические и ручные тесты:
1. **Проверка логина в CRM:**
   - Ввод учётных данных → проверка получения JWT-токена (`accessToken`).
2. **Проверка парсинга всех 3 примеров сообщений:**
   - Ввод текста с опечатками ("вместе", "540т", нумерация `1.`) → проверка правильности выделения магазинов, товаров, цен, количества и совпадения с `catalog_paste.json`.
3. **Проверка Gemini AI парсера:**
   - Ввод ключа Gemini API → запуск ИИ парсинга → сличении результатов с локальным парсером.
4. **Проверка отправки в CRM (`Dry Run` и реальная отправка):**
   - Запуск `--dry-run` → убедиться, что формируется валидный payload для `POST /api/products`.
   - Тестовое создание 1 карточки в CRM и проверка ответа `200 OK`.
